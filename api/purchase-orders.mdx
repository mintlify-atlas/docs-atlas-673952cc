The Purchase Orders API is part of the **purchase-service** running on port `9092`. It exposes purchase orders raised against suppliers. Each order tracks the supplier, the order date, the current approval status, and the total monetary value. The list response is enriched with the supplier's company name, resolved from the **product-service** (which also manages supplier data) via an OpenFeign client call.

<Info>
  Full CRUD for purchase orders is on the roadmap. Currently only the read (list) operation is implemented.
</Info>

---

## Order status lifecycle

Purchase orders move through the following statuses:

| Status      | Meaning                                                                   |
|-------------|---------------------------------------------------------------------------|
| `PENDING`   | Order created, awaiting internal approval                                 |
| `APPROVED`  | Order approved internally, sent to supplier                               |
| `RECEIVED`  | Goods have been received and a goods receipt has been recorded            |
| `CANCELLED` | Order was cancelled before fulfilment                                     |

## Endpoints

### GET /purchase-orders

Returns all purchase orders, each enriched with the supplier's company name.

<Note>
  If the supplier lookup via OpenFeign fails for a given order, `SupplierName` falls back to `"Nombre no disponible"`.
</Note>

**Response fields**

<ResponseField name="purchaseOrderID" type="integer" required>
  Auto-generated primary key of the purchase order.
</ResponseField>

<ResponseField name="purchase_order_supplier_id" type="integer" required>
  ID of the supplier that this order was placed with. References supplier data in product-service.
</ResponseField>

<ResponseField name="SupplierName" type="string" required>
  Supplier company name resolved via OpenFeign call to product-service. Falls back to `"Nombre no disponible"` on error.
</ResponseField>

<ResponseField name="purchase_order_orderDate" type="string" required>
  ISO-8601 datetime when the purchase order was created.
</ResponseField>

<ResponseField name="purchase_order_status" type="string" required>
  Current order status: `PENDING`, `APPROVED`, `RECEIVED`, or `CANCELLED`.
</ResponseField>

<ResponseField name="purchase_order_total_ammount" type="number" required>
  Total monetary value of the order. Stored as `NUMERIC(12, 2)`. Note: the response key name `purchase_order_total_ammount` contains a typo (`ammount`) inherited from the source code.
</ResponseField>

## Data model

| Field          | Type             | Constraints                                                          | Notes                                        |
|----------------|------------------|----------------------------------------------------------------------|----------------------------------------------|
| `id`           | `SERIAL`         | Primary key, auto-generated                                          |                                              |
| `order_code`   | `VARCHAR(20)`    | `UNIQUE`, `NOT NULL`                                                 | Human-readable unique identifier, e.g. `OC-2026-001` |
| `supplier_id`  | `INTEGER`        | `NOT NULL`                                                           | References supplier in product-service       |
| `order_date`   | `TIMESTAMP`      | `DEFAULT CURRENT_TIMESTAMP`                                          | When the order was created                   |
| `status`       | `VARCHAR(20)`    | `CHECK (status IN ('PENDING', 'APPROVED', 'RECEIVED', 'CANCELLED'))` | Current lifecycle state                      |
| `total_amount` | `NUMERIC(12, 2)` | `DEFAULT 0.00`                                                       | Monetary total; updated as line items change |

<Note>
  The `order_code` column (e.g. `OC-2026-001`) is defined in the database schema as a `UNIQUE` identifier but is not currently mapped in the Java entity or included in the API response. It is available directly in the database.
</Note>
