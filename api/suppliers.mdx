---

The Suppliers API manages the vendor records that back the product catalog. Every product must be linked to a supplier, so suppliers are a prerequisite resource. The service also exposes a lightweight name projection endpoint used internally by the purchase-service via OpenFeign for purchase order processing. The service runs on port **9090**, and all paths below are relative to `http://localhost:9090`.

<Info>
  The `GET /supplier/{supplierId}/name` endpoint is consumed internally by the purchase-service Feign client. Avoid changing its response shape without updating the corresponding Feign interface.
</Info>

---

## Endpoints

### GET /supplier

List all suppliers with their full contact and location details.

#### Response fields

<ResponseField name="id" type="number">
  Auto-generated supplier ID.
</ResponseField>

<ResponseField name="companyName" type="string">
  Legal company name of the supplier (max 150 characters).
</ResponseField>

<ResponseField name="contactName" type="string">
  Primary contact person at the supplier (max 100 characters).
</ResponseField>

<ResponseField name="email" type="string">
  Contact email address. Must be a valid email format.
</ResponseField>

<ResponseField name="phone" type="string">
  Contact phone number.
</ResponseField>

<ResponseField name="address" type="string">
  Street address of the supplier.
</ResponseField>

<ResponseField name="country" type="string">
  Country where the supplier is located.
</ResponseField>

<ResponseField name="city" type="string">
  City where the supplier is located.
</ResponseField>

<ResponseField name="status" type="boolean">
  `true` if the supplier is active.
</ResponseField>

<ResponseField name="createdAt" type="string">
  ISO-8601 timestamp of record creation. Read-only.
</ResponseField>

<ResponseField name="updateAt" type="string">
  ISO-8601 timestamp of the last update. Read-only.
</ResponseField>

### GET /supplier/\{supplierId\}/name

Return only the company name for a given supplier ID. This is a read-optimized projection endpoint—it returns a minimal payload to avoid transmitting full supplier records when only the name is needed.

<Note>
  This endpoint is called by the **purchase-service** via an OpenFeign client when building purchase order responses. It is part of the internal inter-service contract and should be treated as a stable API surface.
</Note>

#### Path parameters

<ParamField path="supplierId" type="number" required>
  The ID of the supplier.
</ParamField>

#### Response fields

<ResponseField name="companyName" type="string">
  The supplier's company name.
</ResponseField>

<Tabs>
  <Tab title="Request">
    ```bash cURL
    curl --request GET \
      --url "http://localhost:9090/supplier/1/name"
    ```
  </Tab>
  <Tab title="Response">
    ```json 200
    {
      "companyName": "Tech Solutions Panama"
    }
    ```
  </Tab>
</Tabs>

### POST /supplier

Create a new supplier record.

#### Request body

<ParamField body="companyName" type="string" required>
  Legal company name. Cannot be blank. Maximum 150 characters.
</ParamField>

<ParamField body="contactName" type="string" required>
  Primary contact person name. Cannot be blank. Maximum 100 characters.
</ParamField>

<ParamField body="email" type="string" required>
  Contact email address. Must be a valid email format.
</ParamField>

<ParamField body="phone" type="string" required>
  Contact phone number. Cannot be blank.
</ParamField>

<ParamField body="address" type="string" required>
  Street address. Cannot be blank.
</ParamField>

<ParamField body="country" type="string" required>
  Country of the supplier. Cannot be blank.
</ParamField>

<ParamField body="city" type="string" required>
  City of the supplier. Cannot be blank.
</ParamField>

### DELETE /supplier/\{supplierId\}

Delete a supplier by ID. Returns `204 No Content` on success.

<Warning>
  Deleting a supplier that is still linked to one or more products will cause a foreign key constraint error. Set `status: false` to deactivate the supplier and reassign or deactivate dependent products before deleting.
</Warning>

#### Path parameters

<ParamField path="supplierId" type="number" required>
  The ID of the supplier to delete.
</ParamField>

<Tabs>
  <Tab title="Request">
    ```bash cURL
    curl --request DELETE \
      --url "http://localhost:9090/supplier/3"
    ```
  </Tab>
  <Tab title="Response">
    ```text 204
    (no content)
    ```
  </Tab>
</Tabs>

## Error responses

All errors are returned as an `ErrorDetails` JSON object.

<ResponseField name="timestamp" type="string">
  ISO-8601 timestamp of when the error occurred.
</ResponseField>

<ResponseField name="statusCode" type="number">
  HTTP status code (e.g., `404`, `400`, `500`).
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable error message.
</ResponseField>

<ResponseField name="errorCode" type="string">
  Machine-readable error code. One of `PRODUCT_NOT_FOUND`, `INVALID_REQUEST_DATA`, `VALIDATION_ERROR`, or `INTERNAL_SERVER_ERROR`.
</ResponseField>

<CodeGroup>

```json 404 Not Found
{
  "timestamp": "2024-04-16T09:25:00",
  "statusCode": 404,
  "message": "Supplier not found with id: 999",
  "errorCode": "PRODUCT_NOT_FOUND"
}
```

```json 400 Validation Error
{
  "timestamp": "2024-04-16T09:26:00",
  "statusCode": 400,
  "message": "Incorrect E-mail Format",
  "errorCode": "VALIDATION_ERROR"
}
```

</CodeGroup>
