The Categories API manages the classification taxonomy used throughout the inventory system. Every product must belong to exactly one category, so categories act as a prerequisite resource before any product can be created. The service runs on port **9090**, and all paths below are relative to `http://localhost:9090`.

<Info>
  Category names are enforced as unique at the database level. Attempting to create or update a category with a name that already exists will result in a constraint violation error.
</Info>

---

## Endpoints

### GET /category

List all categories.

#### Response fields

<ResponseField name="id" type="number">
  Auto-generated category ID.
</ResponseField>

<ResponseField name="name" type="string">
  Unique category name (max 50 characters).
</ResponseField>

<ResponseField name="description" type="string">
  Category description.
</ResponseField>

<ResponseField name="status" type="boolean">
  `true` if the category is active.
</ResponseField>

<ResponseField name="createdAt" type="string">
  ISO-8601 timestamp of record creation. Read-only.
</ResponseField>

<ResponseField name="updatedAt" type="string">

### POST /category

Create a new category. The `name` field must be unique across all categories.

#### Request body

<ParamField body="name" type="string" required>
  Unique category name. Maximum 50 characters. Cannot be blank.
</ParamField>

<ParamField body="description" type="string" required>
  Category description. Cannot be blank.
</ParamField>

<ParamField body="status" type="boolean" required>
  Set to `true` to mark the category as active.
</ParamField>

<Warning>
  The `name` field has a `UNIQUE` database constraint. If a category with the same name already exists, the request will fail with a `500 Internal Server Error` caused by a constraint violation. Verify uniqueness before creating.
</Warning>

<Tabs>
  <Tab title="Request">
    ```bash cURL
    curl --request POST \
      --url "http://localhost:9090/category" \
      --header "Content-Type: application/json" \
      --data '{
        "name": "Impresoras",
        "description": "Impresoras láser e inkjet para oficina",
        "status": true
      }'
    ```
  </Tab>
  <Tab title="Response">
    ```json 201
    {
      "id": 5,
      "name": "Impresoras",
      "description": "Impresoras láser e inkjet para oficina",
      "status": true,
      "createdAt": "2024-04-16T09:00:00",
      "updatedAt": null
    }
    ```
  </Tab>
</Tabs>

### DELETE /category/\{categoryId\}

Delete a category by ID. Returns `204 No Content` on success.

<Warning>
  Deleting a category that is still referenced by one or more products will cause a foreign key constraint error. Deactivate the category (`status: false`) and reassign or deactivate dependent products before deleting.
</Warning>

#### Path parameters

<ParamField path="categoryId" type="number" required>
  The ID of the category to delete.
</ParamField>

<Tabs>
  <Tab title="Request">
    ```bash cURL
    curl --request DELETE \
      --url "http://localhost:9090/category/5"
    ```
  </Tab>
  <Tab title="Response">
    ```text 204
    (no content)
    ```
  </Tab>
</Tabs>

## Error responses

All errors are returned as an `ErrorDetails` JSON object.

<ResponseField name="timestamp" type="string">
  ISO-8601 timestamp of when the error occurred.
</ResponseField>

<ResponseField name="statusCode" type="number">
  HTTP status code (e.g., `404`, `400`, `500`).
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable error message.
</ResponseField>

<ResponseField name="errorCode" type="string">
  Machine-readable error code. One of `PRODUCT_NOT_FOUND`, `INVALID_REQUEST_DATA`, `VALIDATION_ERROR`, or `INTERNAL_SERVER_ERROR`.
</ResponseField>

```json 404 Not Found
{
  "timestamp": "2024-04-16T09:20:00",
  "statusCode": 404,
  "message": "Category not found with id: 99",
  "errorCode": "PRODUCT_NOT_FOUND"
}
```
